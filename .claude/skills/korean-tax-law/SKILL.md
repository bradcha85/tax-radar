# 한국 세법 도메인 스킬

Tax Radar 프로젝트의 세금 계산 로직에 적용되는 한국 세법 규칙 레퍼런스.

## CRITICAL: 날짜 기반 검증 프로토콜

**모든 세법 관련 작업 전에 반드시 다음을 수행할 것:**

1. **현재 날짜 확인** — `CLAUDE.md`의 `currentDate` 또는 시스템 날짜를 확인한다
2. **적용 연도 판단** — 현재 날짜가 속하는 과세연도와 부가세 과세기간을 확인한다
3. **일몰 조항 점검** — 아래 표의 `적용기한` 컬럼과 현재 날짜를 비교한다
4. **웹 검색 필수** — 세율·공제율·한도 변경 시, 반드시 최신 세법개정안을 웹 검색하여 교차 검증한다

```
검증 순서:
  현재 날짜 → 적용 과세기간 → 일몰 조항 확인 → 웹 검색 교차검증 → 코드 반영
```

## 1. 부가가치세 (VAT)

### 1.1 기본 세율

| 항목 | 값 | 비고 |
|---|---|---|
| 부가가치세율 | **10%** | 변동 없음 |

### 1.2 의제매입세액공제

#### 공제율 (Credit Rate)

| 대상 | 공제율 | 적용기한 | 코드 위치 |
|---|---|---|---|
| 음식점 개인사업자 (과세표준 4억 이하) | **9/109** | 2026.12.31까지 (특례) | `_getDeemedCreditRate()` |
| 음식점 개인사업자 (과세표준 4억 초과) | **8/108** | 상시 | `_getDeemedCreditRate()` |
| 카페 (비알콜음료점) | **8/108** | 상시 | `_getDeemedCreditRate()` |
| 기타 업종 | **2/102** | 상시 | `_getDeemedCreditRate()` |

> **주의:** 9/109 특례는 일몰 조항이 있다. 현재 날짜가 2026.12.31 이후라면 8/108으로 복귀해야 한다.

#### 한도율 (Cap Rate) — 우대한도율

| 대상 | 과세표준 | 우대한도율 | 적용기한 | 코드 위치 |
|---|---|---|---|---|
| 개인 음식점업 | 1억 이하 | **75%** | 2027.12.31까지 | `_getDeemedLimitRate()` |
| 개인 음식점업 | 1억 초과 ~ 2억 이하 | **70%** | 2027.12.31까지 | `_getDeemedLimitRate()` |
| 개인 음식점업 | 2억 초과 | **60%** | 2027.12.31까지 | `_getDeemedLimitRate()` |
| 개인 기타업종 | 2억 이하 | **65%** | 2027.12.31까지 | `_getDeemedLimitRate()` |
| 개인 기타업종 | 2억 초과 | **55%** | 2027.12.31까지 | `_getDeemedLimitRate()` |

> **우대한도율 일몰 후 (2028~ 예상) 기본한도율:** 음식점 50%/40%, 기타 40%/30%

## 5. 현재 코드 vs 최신 세법 불일치 사항

> 아래는 2026년 2월 기준으로 확인된 불일치 사항이다. 세법 변경 작업 시 우선 처리할 것.

### 5.1 의제매입 한도율 (`_getDeemedLimitRate`)

**현재 코드:**
```dart
if (taxBase <= 200000000) return 0.50;
return 0.40;
```

**최신 세법 (우대한도율, ~2027.12.31):**
```dart
// 음식점업
if (taxBase <= 100000000) return 0.75;
if (taxBase <= 200000000) return 0.70;
return 0.60;
// 기타업종
if (taxBase <= 200000000) return 0.65;
return 0.55;
```

### 5.2 의제매입 공제율 9/109 특례 미적용

**현재 코드:** 음식점·카페 모두 `8/108` 고정
**최신 세법:** 과세표준 4억 이하 음식점은 `9/109` (2026.12.31까지)
