### 2.3 스크린샷 구성 (5장 권장)

앱의 핵심 가치를 전달하는 순서로 배치한다.

#### 1번: 레이더 대시보드 (부가세 금액 강조)

```
화면: 레이더 메인 — 부가세 카드에 예상 금액이 크게 표시된 상태
촬영 조건: 매출 데이터가 입력되어 구체적 금액이 보이는 상태
문구: "다음 부가세, 미리 확인하세요"
강조 포인트: 예상 납부액 숫자를 크게, 납부기한 D-day 배너 노출
```

#### 2번: 의제매입 공제 절감 효과

```
화면: 세금 상세 분석(VAT) — 의제매입세액공제 항목이 절감액으로 표시
촬영 조건: 의제매입 데이터 입력 후 절감액이 표시된 상태
문구: "면세 식재료로 부가세 절감"
강조 포인트: 절감 금액 하이라이트, "공제 전 vs 공제 후" 차이
```

#### 3번: 정확도 게이지 + 데이터 입력

```
화면: 데이터 입력 탭 — 정확도 게이지가 상단에 표시, 입력 카드 목록
촬영 조건: 정확도 60~70% 상태 (일부 입력됨, 개선 여지 보여줌)
문구: "입력할수록 정확해져요"
강조 포인트: 게이지 시각화, "매출 입력 완료 ✓" 상태
```

#### 4번: 세금 캘린더 + D-day

```
화면: 레이더 메인 — 캘린더 영역에 납부기한 이벤트 도트 표시
촬영 조건: 현재 시즌에 맞는 납부기한이 표시된 상태
문구: "납부 기한, 놓치지 마세요"
강조 포인트: 캘린더 날짜에 이벤트 마커, 시즌 배너
```

#### 5번: 시뮬레이터

```
화면: 시뮬레이터 — 매출 슬라이더 조작 중, 세금 변화 실시간 표시
촬영 조건: 슬라이더가 중간 위치, 부가세/종소세 둘 다 표시
문구: "매출이 늘면 세금은?"
강조 포인트: 슬라이더 + 실시간 세금 변화 그래프/숫자
```

### 2.4 스크린샷 디자인 팁

```
1. 배경: 단색 또는 그라데이션 (앱 테마 컬러 #2563EB 계열 또는 #F7F6F3)
2. 디바이스 프레임: 실제 기기 목업에 스크린샷 삽입 (권장, 필수 아님)
3. 문구 위치: 상단 1/3에 문구, 하단 2/3에 앱 화면
4. 폰트: 굵은 고딕 (본고딕 Bold 또는 Pretendard Bold)
5. 문구 크기: 최소 40pt (6.9" 기준) — 썸네일에서도 읽힐 것
6. 첫 번째 스크린샷이 가장 중요: 검색 결과에서 첫 장만 보임
```

### 6.3 Fastfile

**iOS (`ios/fastlane/Fastfile`):**
```ruby
default_platform(:ios)

platform :ios do
  desc "메타데이터 + 스크린샷 업로드"
  lane :metadata do
    deliver(
      skip_binary_upload: true,
      skip_screenshots: false,
      force: true
    )
  end

  desc "TestFlight 업로드"
  lane :beta do
    build_app(
      workspace: "Runner.xcworkspace",
      scheme: "Runner",
      export_method: "app-store",
      output_directory: "../build/ios/ipa"
    )
    upload_to_testflight(
      ipa: "../build/ios/ipa/Runner.ipa",
      skip_waiting_for_build_processing: true
    )
  end

  desc "App Store 심사 제출"
  lane :release do
    deliver(
      ipa: "../build/ios/ipa/Runner.ipa",
      submit_for_review: true,
      automatic_release: false,
      force: true,
      submission_information: {
        add_id_info_uses_idfa: false
      }
    )
  end
end
```

**Android (`android/fastlane/Fastfile`):**
```ruby
default_platform(:android)

platform :android do
  desc "메타데이터 + 스크린샷 업로드"
  lane :metadata do
    upload_to_play_store(
      skip_upload_apk: true,
      skip_upload_aab: true,
      skip_upload_metadata: false,
      skip_upload_images: false,
      skip_upload_screenshots: false
    )
  end

  desc "내부 테스트 트랙 업로드"
  lane :beta do
    upload_to_play_store(
      track: 'internal',
      aab: '../build/app/outputs/bundle/release/app-release.aab',
      release_status: 'completed'
    )
  end

  desc "프로덕션 배포"
  lane :release do
    upload_to_play_store(
      track: 'internal',
      track_promote_to: 'production',
      rollout: '1.0'
    )
  end
end
```

## 8. 스크린샷 촬영 가이드

실제 스크린샷을 찍을 때의 절차:

### 8.1 사전 준비

```
1. 시뮬레이터 설정
   - iOS: iPhone 16 Pro Max (6.9") 시뮬레이터 사용
   - Android: Pixel 8 Pro (1080x2400 → 1080x1920 크롭) 또는 직접 리사이즈

2. 앱 데이터 준비
   - 업종: 한식 일반 음식점
   - 과세유형: 일반과세자
   - 월 매출: 5,000만원 (구체적 금액이 보이도록)
   - 카드비율: 80%
   - 경비: 500만원
   - 의제매입: 1,500만원
   → 정확도 80% 이상 상태로 만들기

3. 시뮬레이터 시간
   - 7월 초순으로 설정 (VAT 1기 시즌 배너 노출)
   - 또는 1월 초순 (VAT 2기 시즌)
```

### 8.2 촬영 순서

```
1. 레이더 메인 화면 → 부가세 탭 선택 상태에서 촬영
2. 부가세 상세 분석 화면 → 의제매입 항목이 보이게 스크롤
3. 데이터 입력 탭 → 정확도 게이지 + 카드 목록
4. 레이더 메인 → 캘린더 영역이 보이게 스크롤
5. 시뮬레이터 화면 → 슬라이더 중간 위치
```

### 8.3 후처리

```
옵션 A (심플): 스크린샷 원본 그대로 제출
  - 장점: 빠름, Apple이 선호 (실제 앱 그대로)
  - 단점: 문구 없어 소구력 약함

옵션 B (디자인): 목업 프레임 + 문구 추가
  - Figma, Canva, 또는 Screenshots Pro 앱 사용
  - 상단 1/3: 홍보 문구 (40pt+)
  - 하단 2/3: 디바이스 프레임 안에 스크린샷
  - 배경: 브랜드 컬러 그라데이션

권장: 옵션 B (전환율 2~3배 차이)
```

## 9. 시즌별 메타데이터 업데이트

스토어 메타데이터는 시즌에 따라 업데이트하여 검색 노출을 극대화한다.

| 시기 | 프로모션 텍스트 변경 | 키워드 조정 | 스크린샷 |
|------|---------------------|------------|---------|
| 12~1월 | "1월 부가세 신고 준비" 강조 | `부가세신고` 추가 | 시즌 배너 포함 촬영 |
| 4~5월 | "5월 종소세 신고 준비" 강조 | `종합소득세신고` 추가 | 종소세 상세 화면 1번으로 |
| 6~7월 | "7월 부가세 신고 준비" 강조 | `부가세계산` 추가 | 부가세 대시보드 1번으로 |
| 비시즌 | 일반 가치 제안 | 기본 키워드 | 기본 구성 |

> iOS 프로모션 텍스트는 심사 없이 변경 가능. 적극 활용할 것.
> Android 짧은 설명도 업데이트 후 즉시 반영.
